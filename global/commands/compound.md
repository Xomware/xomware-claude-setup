# /compound

Capture a pattern, gotcha, or design decision from the current session into a reusable solution doc.

Use the compounder agent to document patterns worth preserving. Invoke mid-session when you learn something that would prevent future mistakes.

1. Parse the instruction — what pattern, gotcha, or decision to document
2. Research the context — read the relevant code, understand why this matters
3. Check `docs/solutions/` for existing docs to update (avoid duplication)
4. Write or update a solution doc in `docs/solutions/[category]/[kebab-name].md`
5. Confirm what was documented
